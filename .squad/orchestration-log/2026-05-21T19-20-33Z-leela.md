### Key Decisions Captured

- **Three-scope model** (Conversation, User, Temp) — automatic key derivation from activity fields
- **Storage abstraction** (IStorage) — simple key-value interface, pluggable backends
- **Lifecycle**: Load before middleware, save after handler
- **Dirty tracking**: JSON hash comparison (language-agnostic, catches nested mutations)
- **Concurrency**: Last-write-wins v1 (optimistic concurrency deferred to v2)
- **TurnContext integration**: `context.state` property (nullable when unconfigured)

### Open Questions Awaiting Rido Approval

1. Middleware vs. built-in: Should `app.UseState(storage)` be a middleware or built into core?
2. Storage config location: Constructor option vs. separate method?
3. Save on handler error: Persist state even when handler throws?
4. Temp scope pre-population: Include `input` (activity.text) automatically?
5. v1 scope: Is MemoryStorage sufficient or need one cloud adapter?
